Trestle — Feature Architecture & Scaling Roadmap

Accelerated path: harden on a 5-10 person pilot team, then jump to 500-1000 users with milestone gates at 100 and 300

1. Planning Model Structure

Workspace Architecture

One shared workspace per year containing 14 plan files:
1 × AOP 12 × RF (monthly) 1 × LTP
Each plan file holds a complete state_data JSON with all input modules. Scenario types: budget forecast actual ltp

AOP — Annual Operating Plan

Full-year budget across all accounts
Exec Summary Budget
Modules: Revenue, HC, Vendor, CTR, T&E, Other, D&A

RF — Rolling Forecast

Monthly re-forecast with actuals split
Exec Summary N+M per month
Labels: 1+11, 2+10, 3+9... (actuals+forecast)

LTP — Long-Term Plan

Multi-year forecast (5 years)
Exec Summary Forecast
Growth assumptions, Y/Y comparisons

5. Accelerated Scaling Roadmap: Pilot → 1000 Users

Compressed to two phases: harden the full stack on a 5-10 person pilot team, then ramp to 500-1000 with milestone testing gates at 100 and 300 users.

Phase 1 — Pilot Hardening (5-10 users, one team)

Scope: standardize the full northstar on a single team

Merged foundation + enterprise work executed end-to-end on one controlled team so every later user lands on a stable base. No feature gaps carried forward.

Auth & Sessions

OAuth (Google / Azure AD) replaces email login · JWT w/ refresh rotation · server-side sessions (express-session + Redis)

Data Model

Normalize state_data JSON into relational tables (employees, vendor_rows, plan_config, field_defs) · row-level PATCH saves · optimistic locking with version column

Access Control & Collab

PostgreSQL RLS policies · dimension-based access (market / bizline / project) · module-level read/write/hidden per role · cell-level CRDT or OT for conflict resolution · DB-persisted audit log · admin user-management UI

Exit criteria to leave pilot

Zero-data-loss under concurrent edit Full close cycle run on Trestle EPM round-trip validated Automated regression suite green
Phase 2 — Scale Ramp (10 → 1000 users)

Infrastructure

Redis cache + session store · read replicas for reporting · WebSocket clustering via Redis pub/sub · rate limiting & request queuing · horizontal app tier behind LB

Integrations

REST API for EPM push/pull · actuals ingestion pipeline · export to Oracle EPM, Anaplan, Workday Adaptive · SSO + SCIM provisioning

Milestone gates (don't open next cohort until green)

~100 users load test at 3× headroom, p95 save < 400ms, RLS audit, disaster-recovery drill
~300 users multi-tenant isolation review, WebSocket fan-out stress, EPM connector reliability SLO, pen test
500-1000 users multi-region read replicas, capacity plan to 2× peak, on-call rotation + runbooks

7. Northstar Structure — Standard vs Custom vs Admin-Managed

Three clearly separated layers so a firm-specific rollout doesn't fork the core product: fixed standard surface, tenant-owned custom fields, and admin-managed field types shared across accounts.

Standard Feature Surface

Core planning modules (fixed)

Revenue, HC & Comp, Vendor/OAO, Contractors, T&E, Other, D&A · monthly proration, bonus/benefits, CapEx split · P&L walk, pivot, scenario compare · multi-user presence, live cursors · XLSX / PNG / clipboard export

Firm-specific EPM integration (standard connectors)

Oracle EPM Anaplan Workday Adaptive SAP / NetSuite GL
Each connector ships as a versioned module with: schema mapping, chartfield crosswalk, incremental + full-load modes, retry/backoff, reconciliation report. New firm = configure, not fork.

Standard dimensions

Business Unit · Biz Line · Market · Project · Account · Pillar · Scenario · Period

Custom Fields (tenant / template)

Template-driven extension points

Each module exposes a custom_fields[] slot defined per tenant template. Fields declared in field_defs with scope (module, row, employee), type, validation, visibility, and default.

Common template needs

HC job family, grade band, equity %, visa status
Vendor contract end, renewal flag, PO #
Revenue deal stage, close probability, product SKU
Contractor agency, SoW id, cost center override

Governance

Custom fields never affect core calc unless opted in · schema migrations per tenant are additive only · exported in pivot/report builder like standard fields

Admin-Managed Field Types (cross-account)

Shared reference data

Firm-admin maintains reusable typed sets surfaced across every plan:
Contractor rates rate cards by role × geo × seniority
Benefits / loaded cost by country / entity
FX rates monthly table, budget vs actual
Salary bands by grade and location
Vendor catalog parent co, tax id, default GL
GL / chart of accounts mirrored from ERP

Propagation

Versioned in reference_sets; plans bind to a version so a rate change doesn't retro-edit a locked close. Admin publishes → tenants pull → users see new defaults on next row create.

Permissioning

Firm-admin write · tenant-admin read + override for their scope · end-user read-only via lookup

8. Scaled & Automated Testing — Feature, Quality, Efficacy

Pilot phase builds the test harness; every milestone gate (100, 300, 500-1000) reruns the full stack against production-shaped synthetic tenants before opening the next cohort.

Feature Testing (does it work)

Layers

Unit calc engine (proration, CapEx, D&A, FX) — ≥90% coverage
Integration module PATCH → DB → WebSocket broadcast
E2E Playwright flows per persona (admin, planner, viewer)
Contract EPM connector schema tests against vendor sandboxes

Automation

Runs on every PR · nightly full-regression on synthetic tenant seed · PR blocked if calc snapshot diffs unexpectedly

Quality Testing (does it hold up)

Load & scale

k6 / Locust scenarios: 100 / 300 / 1000 concurrent editors, month-end save storm, pivot fan-out
SLOs: p95 save < 400ms, p99 WS broadcast < 250ms, error rate < 0.1%

Resilience

Chaos drills (kill primary, drop Redis, partition WS) · DR restore timed at each milestone · cross-tenant isolation fuzz tests

Security

RLS policy test suite · authz matrix tests · dependency & secret scans in CI · external pen test before 300-user gate

Efficacy Testing (does it help)

Calc accuracy

Golden-plan fixtures (AOP, RF, LTP) reconciled to Excel baseline to the cent · EPM round-trip diff must be zero on chartfield × month

User efficacy

Time-to-close telemetry · cells-edited-per-session · error-correction rate · pilot-team NPS baseline and tracked through each milestone

Analytics-driven gates

Feature flags + cohort analytics · A/B on major UX changes · auto-rollback if p95 latency or error rate regresses > 2σ

Test Harness & CI Pipeline

Synthetic tenant generator

Seeds N tenants × M users × realistic plan shapes (14 files, populated modules, reference sets, audit history) for deterministic load and efficacy runs.

Calc snapshot testing

Every PR computes a canonical P&L hash across fixture plans; any unintended drift fails the build with a diff of impacted chartfields.

Environment tiers

dev per-branch ephemeral · staging mirrors prod schema & data volumes · perf dedicated cluster for k6 / chaos runs

Release gates

Unit + integration on PR · E2E + contract on merge · load + chaos + security nightly · full milestone suite before opening each scale cohort (100 → 300 → 1000)
