Current state

  • 14 plans, full module coverage, presence + live cursors WebSocket full-state broadcast
  • Single admin login, email-based access toggles no SSO / SCIM, no row-level security
  • Last edit wins on concurrent saves single-blob state JSON, no cell merge
  • Reference data (rate cards, benefits) maintained informally embedded constants, no versioning
  • Manual import/export to ERP / HR / EPM ad-hoc per integration, not versioned
  • In-session change history only no persisted audit log

Ideal state

  • Same 14-plan model — discrete records per module normalized tables, row-level PATCH, version columns
  • SSO with corporate identity, dimension-scoped access OAuth + SCIM + PostgreSQL RLS policies
  • True cell-level merge — no lost edits per-cell CRDT, Redis pub/sub, optimistic locking
  • Centrally maintained, versioned reference data reference_sets table; plans pin to a version
  • Standard, versioned connectors to ERP / HR / EPM / BI schema map · crosswalk · retry · reconciliation
  • Full persisted change history append-only change_log, monthly partition

Roadmap — Pilot to 1,000 Users

Phase 1 · 5-10 users

Pilot hardening. Lock in every ideal-state capability on one team end-to-end.
Exit: full close cycle on Trestle, zero data loss, green regression suite

~100 users

Open after gate: load test 3× headroom, DR drill, access-control audit
p95 save < 400ms · WS broadcast < 250ms

~300 users

Open after gate: tenant isolation review, concurrent-edit stress, external security review
RLS policy fuzz · pen test · SLO check

500-1,000 users

Multi-region read replicas, capacity to 2× peak, 24/7 on-call with runbooks
multi-AZ failover · queue-backed integrations
SSO / SCIM · single sign-on with corporate identity (Google / Azure AD); SCIM auto-provisions user accounts.
RLS · row-level security in PostgreSQL — restricts which rows a user can read/write based on their dimension scope.
CRDT · conflict-free replicated data type — math-backed structure that lets two users edit the same cell and merge cleanly without conflict.
EPM · enterprise performance management (Oracle EPM, Anaplan, Workday Adaptive) — consolidation and external reporting tools.
