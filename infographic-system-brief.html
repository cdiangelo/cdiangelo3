Trestle — System Brief & Submodule Inventory

First-pass shared reference for technical and finance/finance-tech leads supporting the rebuild. Gauges flag where depth matters; details intentionally shallow — we iterate from here.

1. Exec Summary

Trestle is a multi-user FP&A planning platform: one shared workspace per fiscal year holding 14 plan files (1 AOP + 12 monthly Rolling Forecasts + 1 Long-Term Plan). Each plan file is a state_data JSON containing every input module (revenue, headcount, vendor, contractor, T&E, other, D&A) coded to a shared chartfield grid (BU / Biz Line / Market / Project / Account). Calc outputs feed a P&L walk and a pivot/analytics layer; collaboration runs over WebSocket presence with live cursors and shared state.
The rebuild target preserves this conceptual model but replaces single-blob state with a normalized data spine, layered access control, and standard EPM connectors — see scaling and dataflow briefs for the migration shape.
